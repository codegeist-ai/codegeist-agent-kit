# Chat Rule

Use these rules whenever you create or update `docs/memory-bank/chat.md`.

## Purpose

- Treat `docs/memory-bank/chat.md` as this repo's lightweight memory bank.
- Treat `docs/memory-bank/chat.md` as a compressed working memory for the current project chat.
- Keep it useful for the next session: accurate, compact, and easy to scan.
- Prefer the current truth over a turn-by-turn transcript.

## What Belongs In `docs/memory-bank/chat.md`

- The current project goal and the narrowed scope.
- Decisions that are still in effect.
- What already works in the repo right now.
- Important workflow conventions introduced during the chat.
- Rule or workflow changes that future sessions need immediately.
- Real next steps that are still open.

## What To Remove Or Rewrite

- Obsolete plans that were replaced by a newer decision.
- Repeated discussion that does not add new state.
- Temporary exploration details that no longer matter.
- Raw command output, unless one result materially changes the project state.

## Style

- Write a lively but compact markdown summary, not a full transcript.
- Use short sections, bullets, and small code blocks only for canonical commands.
- Prefer the user's language when it is clear from the chat.
- Keep the file concise enough that someone can understand the current state quickly.
- It is allowed to delete old content when it is no longer accurate or useful.

## Accuracy Checks

- Make sure the summary matches the current repository state.
- When a task is done, move it from future plans into current status.
- Keep next steps realistic and still pending.
- When rules or workflow conventions change, reflect the durable result here.
- If a decision changed, rewrite the old point instead of keeping both versions.
